환불처리 기본 정책

모든 환불처리는 '구매취소 환불' 로 일원화한다.

• 결제 월에 관계 없이 '구매취소 환불' 로 처리한다.

• 상품 타입과 관계 없이 '구매취소 환불' 로 처리한다.

• 결제 수단 및 플랫폼에 관계 없이 '구매취소 환불'로 처리한다.

환불처리 예외케이스 진행방향

1) IAP 구매누락

CS 전용 보너스상품 지급 기능을 제공하여, 고객센터에서 주도적으로 처리할 수 있게 한다.

•CS 전용 보너스 상품 지급 메뉴 : 독립 메뉴 제공하여 별도 히스토리 관리 가능하게 함.  (현재 제공중이 '고객 관리> 보내서 상품지급' 제공 화면과 동일 사양)

•CS 전용 보너스 상품 지급 : '보너스 상품 무료 지급' 과 동일한 프로세스로 제공  (단, 상품은 기존 IAP상품 8개를 적용한다)

•해당 메뉴 권한은 기존 관리자 권한 추가에서 부여할 수 있게 함,

2) 연간상품

연간 상품은 부분취소시, 권한상품/매출상품 이용권만료 처리후, 매출상품은 '구매취소 환불'로 환불 처리한다.

환불 처리일 전/후 환불정보 수정이 필요한 경우, 기존 환불번호로 처리할 수 있도록 관리 기능을 추가한다.

환불처리 상태 : 검토요청, 재검토요청, 검토완료, 환불완료, 최종환불완료

•수정 : 환불정보가 잘못된 경우 환불 정보를 수정할 수 있도록 제공함,

•검토완료 : 요청된 환불처리건을 담당자가 승인할 수 있도록 제공함, (처리자 : 고객센터 황승현님)

•삭제 : 환불요청건이 다른 방식으로 처리되어, 더이상 환불처리가 필요없을 경우 삭제할 수 있도록 제공함,

•환불상태 변경 이력은 확인할 수있도록 히스토리를 제공함.

※ 예외사항

- 환불처리일의 익월 1일 00시 00분부터는 최종 환불처리로 해당건의 수정이 불가능 하기에, 예외적으로 추가 환불접수가 필요한 경우 '수동환불' 기능으로 대체한다.

- 탈퇴/휴면회원의 예외적으로 추가 환불접수가 필요한 경우 '수동환불' 기능으로 대체한다.
