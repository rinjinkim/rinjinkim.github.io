환불처리 기본 정책

모든 환불처리는 '구매취소 환불' 로 일원화한다.

• 결제 월에 관계 없이 '구매취소 환불' 로 처리한다.

• 상품 타입과 관계 없이 '구매취소 환불' 로 처리한다.

• 결제 수단 및 플랫폼에 관계 없이 '구매취소 환불'로 처리한다.

환불처리 예외케이스 진행방향

1) IAP 구매누락

CS 전용 보너스상품 지급 기능을 제공하여, 고객센터에서 주도적으로 처리할 수 있게 한다.

•CS 전용 보너스 상품 지급 메뉴 : 독립 메뉴 제공하여 별도 히스토리 관리 가능하게 함.  (현재 제공중이 '고객 관리> 보내서 상품지급' 제공 화면과 동일 사양)

•CS 전용 보너스 상품 지급 : '보너스 상품 무료 지급' 과 동일한 프로세스로 제공  (단, 구매금액은 환율 영향을 받기 때문에 직접 입력할 수 게 제공한다.)

•해당 메뉴 권한은 기존 관리자 권한 추가에서 부여할 수 있게 함,

연간 상품은 이용권 만료 처리후,  '구매취소 환불' 로 환불처리 할 수 있게 제공한다.

2) 환불처리 수정

환불 처리일 전/후 환불정보 수정이 필요한 경우, 기존 환불번호로 처리할 수 있도록 관리 기능을 추가한다.

환불처리 상태 : 검토요청 ('구매취소 환불' 등록시)  →  재검토요청 ('수정' 시)  →  검토완료 (고객센터 검수완료 후, '검토완료' 시)  → 환불완료 (환불처리일 +1D, 자동처리)

•수정 : 환불정보가 잘못된 경우 환불 정보를 수정할 수 있도록 제공함,

•검토완료 : 요청된 환불처리건을 담당자가 승인할 수 있도록 제공함, (승인1차 : 고객센터 황승현님, 승인2차 : 서운전-김화숙님)

•삭제 : 환불요청건이 다른 방식으로 처리되어, 더이상 환불처리가 필요없을 경우 삭제할 수 잇도록 제공함,

•수정/검토완료된 사항은 처리 내역을 확인할 수있도록 히스토리를 제공함.
